Bii is an awesome dependency manager for C++.

As a C++ developer, I know how annoying it is to hunt down libraries, install them, link them, and pray that you're given a good .obj file. Bii solves this problem, and creates a nice environment for C++ to update and share code. On top of that, Bii has a great community and is stupid-simple.

I just managed to build to build & show an SDL2 Hello World app in just a few minutes with Bii - something that could easily have taken 20-30 minutes (or an hour, if you're like me). The beautiful thing about it is that it's very intuitive. The bii find command is super handy; all you do is #include a block in your .cpp file, and bii will find the corresponding block for you.

Here's an example of building an SDL2 Hello World app with bii (assuming you've got it installed already). This was built on Linux Mint 17 Qiana, using Bii 1.11.

First, bii init SDL2 in a directory. This will create a new folder, SDL2, with "blocks", "bii", and "deps" subdirectories. Now cd into your SDL2 directory, and mkdir -p blocks/<USERNAME>/sdl2. Write a quick main.cpp file in this directory: 


#include "miguel/sdl2/include/SDL.h"
#include <stdio>

const int SCREEN_WIDTH = 640;
const int SCREEN_HEIGHT = 480;

int main(int argc, char* args[])
{
    SDL_Window* window = NULL;

    SDL_Surface* screenSurface = NULL;

    if(SDL_Init(SDL_INIT_VIDEO) < 0) {
        printf("SDL couldn't initialize.");
    }   

    else {
        window = SDL_CreateWindow("SDL Skeleton", SDL_WINDOWPOS_UNDEFINED, SDL_WINDOWPOS_UNDEFINED, SCREEN_WIDTH, SCREEN_HEIGHT, SDL_WINDOW_SHOWN);

        if(NULL == window) {
            printf("Window is null.");
        }

        else {
            screenSurface = SDL_GetWindowSurface(window);

            SDL_FillRect(screenSurface, NULL, SDL_MapRGB(screenSurface->format, 0xff, 0xff, 0xff)); 

            SDL_UpdateWindowSurface(window);

            SDL_Delay(2000);
        }
    }   

    SDL_DestroyWindow(window);
    SDL_Quit();

    return 0;
}
Now, all you have to do is bii find, and bii will locate the SDL2 block for you. Next, do bii cpp:build, and you can see SDL being compiled & linked against your program. The executable will be in SDL2/bin.
